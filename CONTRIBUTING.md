# Contributing

## Before Opening a PR

- Make sure the project can build locally
- Keep changes focused and minimal
- Do not commit API keys, local paths, APKs, or build caches
- Prefer small, reviewable commits

## Code Style

- Follow existing Kotlin and Compose style in the project
- Prefer minimal correct changes over wide refactors
- Avoid introducing compatibility code unless needed

## Reporting Issues

Please include:

- device / Android version
- app behavior
- expected behavior
- reproduction steps
- screenshots or logs when useful

## Sensitive Data

Never include:

- API keys
- personal provider URLs if private
- local SDK paths
- exported user data
