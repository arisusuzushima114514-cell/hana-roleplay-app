Hana

Hana 是一个 Android 平台上的 AI 对话应用，主要围绕角色对话体验来做，当前已经包含角色聊天、多角色群聊、角色卡导入导出、图片生成、长期记忆、联网搜索等功能。

这个仓库是一个适合公开放到 GitHub 的整理副本，已经去掉了本地构建产物、IDE 缓存、本机路径配置以及不适合直接公开的文件。

项目介绍

这是我自己在做的一个个人项目。

更准确一点说，这是我第一次认真去做这一类 App。我的开发基础其实很一般，很多 Android 项目的文件结构、配置方式、构建流程，最开始都不熟。整个过程基本就是一边改、一边在模拟器和手机上测试，再一边继续修。

之所以会开始做这个项目，是因为我本来就对角色设定、人格对话这类产品比较感兴趣。市面上当然已经有很多成熟作品，但我还是想亲手做一个更接近自己想法的版本，哪怕它一开始并不完整。

从最初动手到现在，这个项目是我一点点试出来的。很多地方还在继续打磨，也还有不少不够成熟的地方，但它已经不是一个停留在想法里的东西了，而是真的可以装到手机里、打开、对话、测试、继续迭代的 App。

我把它放到 GitHub，一方面是为了继续记录这个项目的开发过程，另一方面也是想把当前阶段的成果整理出来，方便后续继续维护、补功能，或者和别人交流实现思路。

它现在还只是一个雏形，但已经是一个我会继续认真做下去的项目。

当前状态

项目仍在持续迭代中，功能更新会比较频繁。

v1.7.4 更新内容 (2026-07-17)

【长对话防退化（根治）】
- 每次回复前强制注入角色卡锚点，角色身份始终在模型注意力窗口内
- 锚点四段式结构：内心想法指令 → 角色身份 → 风格样本 → 禁止事项
- 上下文压缩：超过30轮自动摘要旧对话，保留最近30轮完整内容
- 不再出现40轮后角色失忆变人机的问题

【风格样本大幅增强】
- 从对话开头提取高质量回复作为风格参照（前8条，每条400字）
- 额外注入角色开场白作为风格样本（500字）
- 避免人机化回复污染风格，保持角色一致性

【角色对话界面精简】
- 顶部栏精简为单行，冗余功能收入三点菜单
- 底部输入栏紧凑化，缩小圆角和阴影
- 键盘弹出时对话区域不再被挤压

【更新推送系统】
- 新增更新日志弹窗，每次更新后首次打开 App 自动弹出
- "不再显示"选项，同版本不再弹出，下次更新自动弹出新版本日志

【多机型适配】
- 参数面板支持垂直滚动，小屏不裁切按钮
- Slider 流畅度优化，低端设备不卡顿
- 参数保存后即时同步 UI，不再显示旧值

【其他】
- 默认最大回复长度 4096 → 8192 tokens
- 深色模式配色优化

v1.7.2 更新内容 (2026-07-16)

【性能优化】
- 大幅降低单轮对话 token 消耗（约 40-50%），同样余额能聊更多
  - 精简 system prompt 中角色关系指令文本，去除冗余表述
  - 字符对话上下文窗口从 36 条降至 20 条（群聊 20→14）
  - 角色对话中去除不必要的模型版本声明
- 修复开屏动画进入卡顿，淡出过渡更平滑

【多图上传】
- 支持一次选择最多 6 张图片上传
- 上传后显示横向缩略图预览，可单张删除 / 继续添加
- 每张图支持添加文字说明（caption），发送时自动拼入消息

【非视觉模型图片识别】
- DeepSeek / 豆包等不支持视觉的模型，上传图片后自动转为线稿
- 后台静默转换，不增加额外 token 消耗
- 模型可基于线稿轮廓识别图片内容

【Bug 修复】
- 修复角色对话中消息列表莫名滚到顶部的问题
- 修复 <<inner>> 内心独白在中后期不再输出的问题
- 修复发送中自动跟随滚动失效的问题

运行环境

建议环境如下：

- Android Studio
- JDK 17 或 Android Studio 自带 JBR
- Android SDK

项目当前配置大致为：

- minSdk 24
- targetSdk 35
- compileSdk 35

本地构建

Windows:

.\gradlew.bat assembleDebug

macOS / Linux:

./gradlew assembleDebug

构建完成后可生成 Debug 安装包用于本地测试。

配置说明

这个项目包含联网对话、图片生成、模型调用等能力，所以在本地运行时，你通常需要自行准备：

- 可用的模型服务接口
- 对应的 API 密钥
- 需要的话，图片生成服务配置
- 自己测试用的默认参数

为了避免把个人使用环境直接带进公开仓库，建议在发布前反复检查默认配置是否为空、是否有本机地址、是否残留个人密钥或调试信息。

开发建议

结合这个项目当前的功能结构，后续开发时比较建议优先注意下面几件事：

- 联网对话、图片生成、多角色逻辑尽量分别做回归测试
- 发布前检查默认配置和测试账号信息，避免把个人环境带进仓库
- 角色卡 JSON / PNG 的兼容性尽量用多来源样本测试
- 数据导入导出相关功能要留意版本兼容和异常处理
- 如果准备对外长期发布，建议逐步补齐变更记录和已知问题说明
