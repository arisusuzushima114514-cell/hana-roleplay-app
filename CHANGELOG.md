# Changelog

## v1.8.0-beta (2026-07-24)

> 此版本用于公开测试。请反馈个性化、语言切换、开屏裁剪和桌面快捷方式在不同设备与桌面上的实际表现。

### 个性化与聊天显示
- 新增跟随系统、浅色、深色和五组推荐颜色，以及应用内名称、自定义桌面入口、开屏图片和显示时长
- 支持三个底栏图标、AI 与用户气泡图片的独立选择、裁剪、清除与恢复；配置和原图会在覆盖更新后保留
- 新增聊天字号、聊天密度、气泡最大宽度、输入框宽度、头像、消息时间和底栏文字设置，并提供即时预览与标准方案恢复
- 聊天密度在主聊天、角色聊天和预览中统一生效；九宫格边缘保留明确同时作用于上下左右和四角
- 自定义桌面入口会优先更新已有入口，名称可单独更新；系统原始应用图标不支持运行时替换
- 自定义开屏图片改为先进行 9:16 竖屏裁剪并显示保存后的缩略图，不再由后台按屏幕比例临时截取

### 角色聊天与稳定性
- 模型设定、破甲提示与角色面板改为独立入口；角色模型和参数按真实请求优先级保存
- 重新生成仅针对有效的最新助手回复；重置当前对话会恢复开场白并明确重置关系状态
- 为会话、服务商和清空全部对话等危险操作补充确认
- 修复角色会话初始化期间快速发送可能串到主对话、语言切换重建、Prompt 状态检测和生图配置污染问题

### 关系与多角色
- 好感、信任和紧张仅由用户事件与角色公开回应成对确认；内心内容不再直接改写长期关系
- 引用、假设、否定、第三方转述与其他角色台词不会污染当前角色关系
- 多角色公开正文按角色名前缀拆分；未闭合内心标签不会在流式输出时泄漏到正文
- 角色间关系分析忽略私密内心、引用和假设，只记录明确公开的支持、竞争或敌意

### 数据与发布质量
- Room 升级至 schema 16，并兼容数据库半迁移，避免重复字段导致启动崩溃
- 已完成 Debug 单测与 Lint 验证；完整的个性化页多语言覆盖将在后续版本继续补齐

## v1.7.8 (2026-07-20)

### 日语语义辅助
- 当前角色开启有效破甲提示后，自动启用单请求日语语义辅助，与普通创作预设严格隔离
- 短消息使用轻量核对，复杂剧情自动进行日语到中文的完整双重语义校验
- 核对人物主体、动作对象、否定、条件、时间顺序、称呼、关系和全部剧情节点
- 日语语义核对后继续融合角色身份、关系阶段、历史、共同场景和角色间关系
- 原始中文始终是唯一事实依据，最终只输出自然中文，不显示或保存日语中介文本

### 执行与成稿校验
- 新增中文成稿校验，检查日语残留、串角色、动作对象错误、否定反转和剧情遗漏
- 新增 system 末端执行优先级锁，减少 Flash 在长上下文中跳过破甲与语义规则
- 最终 Prompt 可查看轻量/完整模式、角色状态融合、中文成稿校验和末端执行锁状态
- 修复 Prompt 预览没有使用本轮用户原文，导致显示模式与真实请求不一致的问题

### 诊断与兼容
- 新增日语中介残留、简短拒绝模板和异常短正文诊断
- 识别 `content_filter`、`SAFETY`、`blocked` 等上游拦截原因
- 强烈中文词汇的输入和返回链路保持原文，不进行本地删除、替换或弱化
- 真正断线时继续保存已经生成的部分回复
- 新增自动化回归测试，覆盖破甲隔离、强烈词汇保真、拒绝诊断和会话分类

### 主对话隔离修复
- 修复群聊和故事仍出现在主对话抽屉、搜索与收藏夹的问题
- 修复启动恢复或返回主对话时可能错误选中群聊或故事的问题
- 主对话长期记忆不再注入或提取群聊、故事与角色对话内容
- 兼容旧格式群聊，只要存在参与角色就不会被归入主对话

### 群聊认知与输出修复
- 所有群聊角色从会话建立开始就明确知道彼此共同在场，不再等用户询问后才承认看得见对方
- 每轮注入所有在场角色的公开身份摘要，并扩大最近公开群聊记录范围
- 修复单个角色绑定失效模型时独自返回 404：请求前校验并回退，实际 404 时再自动重试一次默认模型
- 清理当前角色重复名字、括号名字和连续前缀，删除模型替其他角色输出的“角色名：台词”行
- 如果模型整段都在替其他角色回答，将视为空回复而不是保留错误的多角色输出

### 高好感关系连续性修复
- 高好感、高信任和高接受度时，角色会持续保持熟悉、撒娇、依赖、调情与自然靠近，不再无理由退回陌生人语气
- 没有威胁、胁迫、暴力、明确拒绝或撤回同意时，不再凭空把既定亲密互动解释为非自愿或性暴力
- 高好感不等于永久同意；本轮明确拒绝、迟疑或撤回仍会被尊重，并以符合角色人设的方式表达边界
- “我无法处理涉及非自愿……”等模型/上游元安全模板不再更新好感、信任、紧张或关系锚点
- 元安全拒绝不会继续注入后续上下文，避免角色长期模仿政策式拒绝并突然变得生疏
- 关系连续性 Prompt 改用“双方成年、清楚表达、相互愿意”的正向表述，不再主动枚举容易触发服务商提交审核的敏感类别
- Google 返回的英文 Prompt 拦截信息现在会被正确识别为上游策略拦截，并给出明确诊断

### 生成稳定性与消息操作修复
- 修复角色页顶部“重新生成”错误选中最早助手消息、导致后续对话几乎全部被删除的问题；现在只重新生成最新助手回复
- 设置面板中的重新生成同步改为最新助手回复
- 将不直观的“撤回最后一轮”入口替换为带确认框的“清空对话”
- 精简破甲联动的日语语义内部检查，短消息不再执行重复回译和多轮事实清单，降低首字等待时间
- 识别 `finish_reason=length` 和输出 token 上限截断，保留已生成正文并明确提示提高最大输出或生成较短版本
- 群聊遇到输出长度上限时同样保存已有正文，不再表现为回复突然消失
- 角色页右上角三点菜单原“重新生成”入口已直接替换为红色“清空对话”，不再并列显示两个容易混淆的操作
- 清空前必须通过确认弹窗；弹窗明确说明聊天记录、好感度、信任度、接受度、关系阶段和关系时间线均会重置

### 群聊收纳与角色预设标记
- 群聊页面改为与角色卡一致的单行顶部布局，只保留返回、标题和三点菜单，减少顶部区域遮挡聊天视野
- 群聊参与角色、共同场景、角色关系图、Prompt、背景和重命名统一收入右上角三点菜单
- 角色宇宙页面关闭主对话抽屉手势，不再让群聊看起来属于主对话导航
- 主对话页面从入口层只接收主对话会话、消息和流式状态，群聊不会再短暂或错误显示在主对话中
- 旧格式群聊只要保存了参与角色列表，也会被正确收纳到角色宇宙的“群聊”分区
- 角色卡列表头像旁新增独立双标记，可同时显示“破甲”和“角色预设”是否开启，并区分是否允许影响人格

### 单角色卡多角色第一版
- 新增“卡内角色”编辑区，可在一张角色卡中添加、编辑和删除明确的子角色资料
- 旧角色卡默认保持单角色模式，只有用户明确添加或导入 metadata 中的多角色结构时才启用，不根据正文猜测
- 酒馆 PNG/JSON metadata 中明确存在 `subCharacters`、`subCharacterProfiles` 或 `characters` 数组时会保留；明确的根 JSON 角色数组则以第一个为主角色、其余作为子角色
- Prompt 注入主角色和子角色名单，要求使用 `<sub_character name="角色名">` 与 `<inner character="角色名">` 标签归属内容
- 一条回复仍作为一个消息气泡保存，子角色公开内容会在气泡内部按角色名分段显示，内心想法按角色分组展开
- 角色列表新增“多角色·N”徽标，肉眼可见区分单角色卡与多角色卡
- 独立 JSON、酒馆 PNG 和全量备份导出均保留多角色结构
- Room 升级至 schema 16，旧数据使用默认值无损迁移

## v1.7.7 (2026-07-20)

### 群聊全面升级
- 每条用户消息只触发一轮群聊，动态选择 1~3 个角色依次回复
- 后续角色可以引用本轮上一位角色，支持吃醋、竞争、反驳、附和和自然接话
- 新增微信式引用气泡，并持久化群聊回合与引用关系
- 角色间互动与用户关系状态隔离，不再污染用户好感、信任和关键互动
- 新增有向角色关系图，长期记录角色之间的亲近、竞争、紧张和最近事件
- 新增共同场景编辑器，可设定地点、时间、氛围和事件规则
- 新增强制同场模式，禁止角色自行离场、跳转地点或把其他角色写出场
- 新增群聊认知隔离，其他角色无法读取 `<inner>`、reasoning、隐藏计划或未公开意图
- 引用消息只保存公开正文，避免私密内心通过引用泄漏
- 群聊顶部显示共同场景摘要与锁定状态，点击即可快速编辑

### 角色提示独立
- 原全局创作预设改为「破甲提示」，已有文本和角色开关自动保留
- 每张角色卡新增独立创作预设文本、应用开关和「允许影响人格」开关
- 角色独立创作预设与破甲提示可以分别或同时启用
- 修改某张角色卡的正常预设不再影响其他角色
- 最终 Prompt 预览分别显示角色创作预设与破甲提示的实际注入状态

### 角色宇宙导航
- 群聊创建成功后直接在角色宇宙内打开，不再跳转到主对话
- 故事创建和打开后同样保留在角色宇宙
- 群聊和故事新增独立返回入口，返回后回到对应列表

### Flash 稳定性
- 高频流式小分片改为短窗口批量刷新，降低高速生成时的界面重组压力
- 兼容返回 `finish_reason` 但省略 `[DONE]` 的 OpenAI 兼容接口
- 真正断线时仍保存已生成的部分回复，减少回复生成到一半后丢失

### 主题与气泡
- 新增紫罗兰、海水蓝、樱花粉、森林绿和落日橙五套主题色
- 每套主题提供浅色与深色配色，并支持跟随系统
- 主对话、群聊、故事和角色卡统一使用主题气泡颜色
- 不同主题拥有差异化气泡圆角和尾角，外观设置页支持即时预览
